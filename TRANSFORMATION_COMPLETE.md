### Data Flow Examples

**1. User Upgrades to Pro Tier:**
```
User clicks "Upgrade to Pro"
  ‚Üí POST /api/billing/create-checkout { tier: 'pro' }
  ‚Üí StripeService.createCheckoutSession()
  ‚Üí Stripe creates checkout session
  ‚Üí User redirected to Stripe Checkout
  ‚Üí User completes payment
  ‚Üí Stripe sends webhook: checkout.session.completed
  ‚Üí POST /api/billing/webhook
  ‚Üí handleCheckoutComplete()
  ‚Üí Update User: tier='pro', status='active', quotas set
  ‚Üí User redirected to success page
  ‚Üí Dashboard shows Pro quotas
```

**2. User Generates Code:**
```
User submits code request
  ‚Üí Middleware: checkUsageQuota('codeGenerations')
  ‚Üí Check: usageCount.codeGenerations < usageQuota.codeGenerations
  ‚Üí If OK: next()
  ‚Üí If exceeded: 429 Too Many Requests
  ‚Üí CodeGenService.generateCode({ description, language })
  ‚Üí OpenAI API call
  ‚Üí Response: { code, explanation }
  ‚Üí incrementUsage(userId, 'codeGenerations')
  ‚Üí Return code to user
```

**3. Monthly Usage Reset:**
```
Cron job runs monthly (1st of month)
  ‚Üí Find all users with lastReset < 30 days ago
  ‚Üí For each user:
      ‚Üí usageCount = { messages: 0, images: 0, ... }
      ‚Üí lastReset = new Date()
      ‚Üí user.save()
  ‚Üí Log: "Reset usage for {count} users"
```

// Find users by subscription tier
db.users.find({ subscriptionTier: 'pro' })

// Find users exceeding quota
db.users.find({
  $expr: { $gte: ['$usageCount.messages', '$usageQuota.messages'] }
})

// Count users per tier
db.users.aggregate([
  { $group: { _id: '$subscriptionTier', count: { $sum: 1 } } }

// Find users with expiring subscriptions (next 7 days)
db.users.find({
